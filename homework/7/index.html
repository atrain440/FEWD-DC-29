Homework 7 - Relaxr Responsive Site

Description

Relaxr's user count has grown, and the number of users accessing the site from a mobile device has surprised everyone, even by today's standards. The product managers at Relaxr have asked you to return to the project and make their site responsive so it renders nicely on mobile devices. They had one of their internal developers make a few tweaks to your source code, and they have given you the new HTML / CSS files to work from. Using these starter files, build out the mobile layouts for the landing page and blog.

Files for this assignment

homework-7.zip

Due date for this assignment

January 9th, 2017 (Monday)

Real-World Applications

  • Use media queries to render different CSS files according to screen size
  • Revisit applications and projects to iterate on and improve code you've written

Technical Requirements

  • Use media queries at 480px, 768px, 960px and 1200px breakpoints
  • Make the content of the site adjust at each breakpoint, even if it's slight - make something happen.
  • As a user changes the size of a browser window:
  • Content remains in appropriate divs
  • Content remains visible to the user
  • Fonts change size appropriately
  • Margins and padding change size appropriately
  • Content does not overlap
  • In the blog section, the two-column layout changes to a single-column layout when appropriate

Submission

  • You will submit this homework assignment to us via Github. If you cannot figure out Github just send it via Slack / .zip file method we have been using for other assignments.

Evaluation

The instructional team will grade each technical requirement and provide a numeric grade on a scale: does not meet expectations (0); meets expectations (1); exceeds expectations (2). The maximum possible score on this assignment is 12/12. You will be evaluated on 6 key items being completed in this assignment:

  • Use of four breakpoints
  • HTML does NOT change as breakpoints are hit
  • Margins and padding are adjusted to create appropriate readability
  • Blog goes from two to one columns when it needs to
  • Fonts change when necessary, readability is key here
